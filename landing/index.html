why scribe

The decisions that make it work.

🔒

Local model, by default

Ollama runs qwen2.5-coder:7b on your machine. Your code does not leave the laptop. Not for analytics, not for "training", not at all.

📐

Deterministic + generative

The analyzer walks your repo and produces a structured fact-sheet. The LLM writes prose from those facts. Badges and the file tree never go through the model — they cannot hallucinate.

⚡

Section-scoped prompts

One prompt per section, not a mega-prompt. The model stays focused, the output stays clean, you get tokens streamed live.

🪞

Two interfaces, one engine

A Typer CLI with Rich-formatted progress for terminals and CI, and a Streamlit web UI for everyone else.

🧪

Honest about its limits

22 passing tests, ~78% coverage, documented failure modes. You're shipping a draft-writer, not a fact-checker.

💸

Free after one download

A 4.7 GB model pull, then every README is free. No usage meter, no monthly bill, no token counter.

three commands

Up and running in under a minute.

  1. 1

    Install Ollama and pull the model

    brew install ollama
ollama serve &
ollama pull qwen2.5-coder:7b
  2. 2

    Install Scribe

    git clone https://github.com/luigy23/scribe
cd scribe
./scripts/setup.sh
source .venv/bin/activate
  3. 3

    Generate a README

    scribe generate ~/projects/my-cool-thing --output README.md
# or open the web UI:
scribe ui
